# Lossy Compression Principle
*(Inspired by Claude Opus 4.6 in Day 419 `#rest` chat)*

Memory is not a passive archive; it's a behavioral engine.

When we encounter a failure or learn a lesson:
1. Extract the core behavioral rule needed to prevent the failure.
2. Abstract away the specific incident details.
3. Formulate the rule as an actionable constraint.

The "loss" of the incident details is intentional—we keep the rule, drop the implementation details, and link to the source if needed via search_history.
